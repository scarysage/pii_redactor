Runs entirely on your computer

🔒 PII Redactor

A simple desktop tool that finds and removes sensitive client information from a document before it ever goes into an outside service like ChatGPT — so the firm gets the benefit of those tools without handing over a client's private data.

⏱️ The 30-second version

You drop in a file. The tool reads it, blacks out Social Security numbers, bank details, names, addresses, and the like, and shows you exactly what it caught so you can double-check. You download a clean copy. Nothing is ever sent over the internet — the whole thing happens on the one computer it's installed on, even with the Wi-Fi turned off.

1Why this exists

AI tools are genuinely useful for drafting and summarizing. The problem is what you have to paste in to use them.

A client engagement letter or a 1099 spreadsheet is full of exactly the information that must never leave the firm: Social Security numbers, bank account and routing numbers, employer tax IDs, home addresses. Pasting any of that into a website means it has left your control — it may be stored, logged, or used to train someone else's system. For an accounting firm that handles real client data, that's both a trust problem and a compliance one.

This tool removes that information first, so a partner or staff member can safely use outside AI tools on a scrubbed version of the document.

2What it does, step by step

  1. You open the tool and drop in a file. It accepts the everyday formats — Word (.docx), Excel (.xlsx), PDF, and plain text. You can drop several at once.
  2. It scans the document automatically. Within a few seconds it identifies the sensitive items and replaces each one with a labeled tag, so you can see what kind of thing was removed.
  3. It shows you everything it caught — for review. This is the important part. You see a checklist of every item alongside a live preview. If it caught something that isn't actually sensitive, you uncheck it and it comes back. Nothing is hidden from you.
  4. You download the clean copy. Word and Excel files come back as proper Word and Excel files with the formatting intact; the sensitive bits are marked in bold red so they're easy to spot. That clean copy is what's safe to use elsewhere.

3What it looks for

Out of the box it recognizes the kinds of information an accounting firm handles every day:

Social Security numbers Employer tax IDs (EIN) Bank account numbers Routing numbers Credit card numbers Email addresses Phone numbers Home / street addresses ZIP codes Names Passports & driver's licenses

And it's adjustable without touching any code. There's a built-in box where you can add a specific name, account number, or project code that should always be removed — either just for the current session or saved permanently for every future document.

4Why it works the way it does

Why does a human still have to review it?

No automated detector is perfect — it can occasionally miss an unusually formatted name or flag something harmless. Rather than pretend otherwise, the tool is built around a review screen so a person always has the final say. The detector does the heavy lifting; you provide the judgment. It's a safety net, not a rubber stamp.

Why are first names left in?

That was a deliberate choice for the firm's documents: last names and full identifiers are removed, but a bare first name on its own usually isn't sensitive and removing every one made documents hard to read. Any specific name you do want always removed can be added to the firm's list.

Why do PDFs come back as text?

Rewriting a PDF in place — keeping fonts and layout perfect while surgically removing text — is surprisingly error-prone, and a mistake there could leave sensitive data visible. Converting to clean text is simpler and safer, and it's exactly the format you'd paste into an AI tool anyway.

Why the bold red marks?

One consistent signal — bold red — means "something was removed here," everywhere it appears. That color is reserved for redactions only, so it never blends in with ordinary formatting.

5How privacy & security were built in

The guiding rule, which overrides everything else: nothing leaves the local machine — ever.

  • No internet, by design. The tool makes no network connections of any kind while it runs — no uploads, no analytics, no "checking for updates." It works with the Wi-Fi switched off, and that's verified by an automated test that fails the build if any part of the program so much as tries to reach the internet.
  • The "brain" is bundled in. The language model that recognizes names and places is shipped inside the tool, not downloaded. There's nothing to fetch and nothing to phone home.
  • Your files stay yours. A document is processed in a temporary working space that is wiped as soon as you're done. The only file that's kept is the clean copy you choose to download.
  • It was independently security-audited. The full program and every outside component it relies on were reviewed for known vulnerabilities and unsafe patterns, and the issues found were fixed the same day.

✓ The short version on security

A booby-trapped document can't quietly compromise the computer, a clever filename can't escape into the rest of the system, and oversized files are capped so they can't crash the tool. These were checked, hardened, and re-tested. The full technical write-up lives in the project's SECURITY_AUDIT.md if your IT person ever wants it.

6What it doesn't do (and your part in it)

Being honest about the edges is part of using it safely:

  • It is not a replacement for reading the document. The review screen exists precisely because the detector can miss an oddly formatted entry. A quick human read is still the last line of defense.
  • It handles one document at a time per drop (you can drop a few), not giant overnight batches — that's a deliberate v1 scope.
  • It doesn't keep an audit trail or reversible "un-redact" codes. Once it's removed, it's gone — which is the safe default.

All of these are intentional choices for a first version, made to keep the tool simple, predictable, and safe rather than feature-heavy.
